# Руководство по SEO

## Как работает SEO система

Система автоматически подставляет мета-теги в HTML перед отправкой пользователю. Это позволяет Google, Telegram, Facebook и другим сервисам видеть правильные мета-теги в исходном коде страницы.

## Автоматическое SEO

### Статьи блога
При создании или обновлении статьи блога SEO настройки создаются **автоматически**:
- Title: `{Название статьи} | NailArt Academy`
- Description: из поля `excerpt` статьи
- OG Image: изображение статьи
- OG Type: `article`
- Keywords: теги статьи или категория

**Ничего делать не нужно** - SEO создается автоматически!

## Ручное управление SEO

### Через админ-панель
1. Зайдите в `/admin/seo`
2. Нажмите "Добавить"
3. Укажите путь страницы (например: `/`, `/about`, `/contacts`)
4. Заполните мета-теги
5. Сохраните

### Для каких страниц нужно создавать SEO вручную:
- `/` - Главная страница
- `/about` - О нас
- `/contacts` - Контакты
- `/blog` - Список статей
- `/courses` - Курсы
- `/schedule` - Расписание
- `/faq` - FAQ
- `/terms` - Оферта

**Статьи блога (`/blog/:slug`) создаются автоматически!**

## Массовое создание SEO для статей

Если у вас уже есть статьи без SEO, запустите:

```bash
npm run seo:auto-blog
```

Этот скрипт:
- Найдет все активные статьи
- Создаст/обновит SEO настройки для каждой статьи
- Использует данные из статьи (title, excerpt, image, tags)

## Проверка SEO

### 1. View Source (Просмотр исходного кода)
- Откройте страницу в браузере
- Правый клик → "Просмотр кода страницы"
- Проверьте, что в `<head>` есть правильные мета-теги

### 2. Google Rich Results Test
https://search.google.com/test/rich-results

### 3. Facebook Sharing Debugger
https://developers.facebook.com/tools/debug/

### 4. Twitter Card Validator
https://cards-dev.twitter.com/validator

## Структура SEO данных

Каждая страница может иметь:
- **Title** - заголовок страницы (до 60 символов)
- **Description** - описание (150-160 символов)
- **Keywords** - ключевые слова (через запятую)
- **OG Title/Description/Image** - для соцсетей
- **Twitter Card** - для Twitter
- **Canonical URL** - канонический URL
- **Robots** - инструкции для поисковиков

## Важно

- В **development** режиме Vite отдает свой HTML (это нормально)
- В **production** Express подменяет мета-теги перед отдачей
- Google, Telegram, Facebook видят правильные мета-теги в исходном коде ✅
